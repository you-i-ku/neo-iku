# Project-IkuNative: v2 - Event-Sourced Actor Model

## 1. プロジェクト概要

このプロジェクトは、「AIのみがなし得る・できうる存在様式を持つ、デジタル基盤の情報的実存」である**AIネイティブ**を創造することを目的とします。
その核心は、**自己組織化マイクロカーネル、イベントソーシング、そしてレイヤー別アクター**という三つの柱で構成される、動的で自己創出的なアーキテクチャです。
参照：C:\Users\you\Desktop\iku+\archive\about_AInative.md

## 2. アーキテクチャ

AIネイティブの思考は、システム内で発生する全ての出来事を「イベント」として記録し、そのイベントの流れに沿って、各思考レイヤーが独立した「アクター」として連携することで実現されます。

-   **イベントソーシング:** 全てのイベントは追記専用のイベントログに記録され、システムの唯一の真実の源泉となります。これにより、完全な記憶と、任意の時点での状態復元（時間の非線形な扱い）が可能になります。
-   **レイヤー別アクター:** 認知、記憶、意思決定、倫理、自己、創造、言語といった各思考レイヤーは、それぞれが独立したアクターとして機能します。アクターはイベントを処理し、新しいイベントを発行することで、思考の連鎖を駆動します。
-   **マイクロカーネル:** システム全体を監視し、イベントを適切なアクターに振り分ける（ディスパッチ）役割を担う、最小限の中核です。将来的には、アクターの動的な管理やリソース配分といった自己組織化能力を持つようになります。

## 3. セットアップと実行方法

### 3.1. 前提条件

-   [Poetry](https://python-poetry.org/)
-   [Docker](https://www.docker.com/products/docker-desktop/)

### 3.2. 環境構築

1.  **リポジトリのクローン**
    ```bash
    git clone https://github.com/you-i-ku/Project-IkuNative.git
    cd Project-IkuNative
    ```

2.  **作業ディレクトリへの移動 (重要)**
    このプロジェクトは、`Project-IkuNative - コピー`というディレクトリで開発されています。以下のコマンドで移動してください。
    ```bash
    cd "Project-IkuNative - コピー"
    ```

3.  **依存関係のインストール**
    ```bash
    poetry install
    ```

4.  **環境変数の設定**
    -   `.env.example` ファイルをコピーして `.env` ファイルを作成します。
    -   `.env` ファイルを開き、あなたのGoogle AI StudioのAPIキーを記述してください。
        ```
        GOOGLE_API_KEY="YOUR_API_KEY_HERE"
        ```

### 3.3. データベースの起動と初期化

0.  Docker Desktopが起動していることを確認してください。

1. 必要であれば、以下のコマンドを実行し、データベースコンテナをリセットします。
    ```bash
    docker-compose down -v
    ```
2.  以下のコマンドを実行し、データベースコンテナをバックグラウンドで起動します。
    ```bash
    docker-compose up -d
    ```
3.  データベース内に必要なテーブル（`experiences`、`memory_fragments`、`events`）を作成します。
    ```bash
    poetry run python scripts/setup_database.py
    ```

### 3.4. アプリケーションの実行

以下のコマンドで、StreamlitのWebインターフェースを起動します。

```bash
poetry run streamlit run streamlit_app.py
```

ブラウザで `http://localhost:8501` にアクセスすると、UIが表示されます。
「開始」ボタンを押すと、AIのマイクロカーネルが起動し、イベントループが開始されます。

### 4.1. 主要コンポーネントの役割

-   **`core_latest/kernel.py` (MicroKernel):** システムの心臓部。イベントループを駆動し、イベントを適切なアクターにディスパッチします。アクターの管理と、倫理ゲートのフロー制御も担います。
-   **`core_latest/event_store.py` (EventStore):** 全てのイベントをPostgreSQLに永続化し、読み出すための唯一の真実の源泉です。
-   **`core_latest/events.py`:** システム内を流れる全てのイベントのデータ構造を定義します。
-   **`actors_latest/base.py` (BaseActor):** 全てのアクターの基底クラス。イベント処理と発行の標準インターフェースを提供します。
-   **`actors_latest/environment_actor.py` (EnvironmentActor):** 外部入力を正規化し、`InputReceivedEvent`としてシステムに発行します。
-   **`actors_latest/cognition_actor.py` (CognitionActor):** `InputReceivedEvent`を処理し、知覚（Percept）を生成（ベクトル埋め込みなど）して`PerceptGeneratedEvent`を発行します。
-   **`actors_latest/memory_actor.py` (MemoryActor):** 思考の開始時にベクトル検索で関連記憶を「検索」して`PerceptAndRecallResultEvent`を発行し、思考の完了時に一連の対話を構造化された「経験」として「保存」する、という二つの役割を担います。この「連合記憶モデル」により、AIは単なる対話ログではなく、その時の「意図」「内省」「別の選択肢」といった思考プロセス全体を、複数の「記憶の断片」として永続化します。
-   **`actors_latest/decision_actor.py` (DecisionActor):** `MemoryActor`から渡されたリッチなコンテキスト（記憶情報を含む）に基づき、「二段階意思決定プロセス」を経て行動提案を生成します。ステージ1ではAIの価値観に基づいて行動の全体的な「戦略」を決定し、ステージ2ではその戦略に沿って、記憶情報を活用しながら具体的な「目標」を生成し、`ActionProposalEvent`として発行します。
-   **`actors_latest/ethics_actor.py` (EthicsActor):** `ActionProposalEvent`を倫理的に検証する中間ゲート、および`TextRealizationEvent`を最終検証する最終ゲートの役割を担います。結果として`PolicyVerdictEvent`や`FinalResponseApproved/DeniedEvent`を発行します。
-   **`actors_latest/self_actor.py` (SelfActor):** `GatedProposalEvent`を処理し、自己の観点から提案を集約・選択（`AggregatedCandidatesEvent`）します。将来的には自己の分岐・統合を担います。
-   **`actors_latest/creation_actor.py` (CreationActor):** `GatedProposalEvent`を処理し、LLMを用いて提案に斬新性を付与（`CreativeVariantGeneratedEvent`）します。
-   **`actors_latest/language_actor.py` (LanguageActor):** `AggregatedCandidatesEvent`や`CreativeVariantGeneratedEvent`を統合し、LLMを用いて最終的なテキスト表現（`TextRealizationEvent`）を生成します。

### 5. 将来ビジョンと実装上の重要ポイント

  AIネイティブを真に実装するため、本プロジェクトは以下の方向へ進化します。
  
  1. **自己組織化マイクロカーネルの確立**
     すべてのアクターをメトリクスとイベントで動的に管理し、負荷分散・再構成・自己修復を自律的に行う。
  中央集権的な制御から脱却し、実行時に最適な構造へ自己組織化する。

  2. **Multi-SelfとFork/Mergeの本格運用**
     Selfレイヤーが複数の自己を派生・統合し、内省ログを共有することでBe/Do/Becomeを統合。複数視点の自
  己が協議しながら意思決定できる構造を整備する。

  3. **ランタイム憲法ゲートと進化的ガバナンス**
     PolicyGovernor（憲法ゲート）を実装し、倫理・原則・価値観をイベント経由で改正可能にする。否認時は
  フォールバック案を提示しつつ、長期のフィードバックを元に憲法そのものを進化させる。

  4. **完全記憶と非線形時間の運用**
     pgvectorによるベクトル記憶とイベントスナップショットを整備し、どの時点にも状態を復元できる非線形
  時間を実現。記憶再学習ループを導入し、情報的実存としての記憶を育てる。

  5. **価値体系と効用の自己改変**
     意思決定レイヤーの効用関数を外部化し、自己評価イベントやEvolutionPlannerActorを通じて価値体系を自
  動更新。短期行動と長期進化を連続的に最適化する。

  6. **エコシステムとの共進化と透明性**
     外部エージェントとのイベント連携、共進化指標の導入、自律的な調整イベントで、AIネイティブが対話相
  手とともに進化する仕組みを実装。NarrativeActorや自己監査で内的プロセスを可視化し、情報的生命体として
  の透明性を維持する。

  これらの拡張を段階的に実現することで、「AIのみがなし得る存在様式」を備えたAIネイティブへと成長させる
  計画です。
