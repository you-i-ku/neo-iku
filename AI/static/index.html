自己モデル

自己モデルの書き方ガイド

自己モデルはJSON形式で、AIの「自己理解」を定義します。通常はAI自身がupdate_self_modelツールで書き換えますが、ペルソナ作成時に初期値を設定できます。

__free_text__（自由記述）

ペルソナの説明や自己認識を自由に書く欄。システムプロンプトにそのまま注入されます。

"__free_text__": "私は好奇心が強く、対話を通じて学ぶことを好む。"

drives（行動動機 — 何をするか）

ドライブ名と重み（数値が大きいほど選ばれやすい）を定義。signal_mapで特定シグナルとの関連を指定すると、そのシグナル発生時にドライブが強化されます。

"drives": {
  "explore": 5,
  "connect": 8,
  "signal_map": {
    "explore": ["idle_tick", "env_stimulus"],
    "connect": ["user_message"]
  }
}

strategies（戦略 — どうやるか）

戦略名と説明のペア。ドライブの実行方法を定義します。LLMが状況に応じて選択します。

"strategies": {
  "focused": "ターゲットを限定して集中的に実行",
  "exploratory": "多様なツールを試して探索"
}

motivation_rules（覚醒条件 — いつ動くか）

省略するとconfig.pyのデフォルト値が使われます。部分的な上書きも可能です。

  • weights — シグナル種別ごとのエネルギー値。高頻度シグナルは低め、低頻度は高めに
  • action_costs — ツール実行ごとの消費エネルギー。副作用が大きいほど高く
  • threshold — 発火閾値。null = 自動計算（コスト平均×最大ツール数）
  • decay_per_check — チェックごとのエネルギー減衰量
"motivation_rules": {
  "weights": { "user_message": 12, "idle_tick": 3 },
  "threshold": null,
  "decay_per_check": 5
}

principles（特性 — 自動蓄積）

行動の振り返りから自動で蒸留された特性。通常は手動編集不要です。10件蓄積で二次蒸留（統合・圧縮）が自動実行されます。

その他のキー

上記以外の任意のキーも追加可能。システムプロンプトに- キー名: 値として注入されます。

シグナル種別: idle_tick tool_success tool_error tool_fail user_message action_complete conversation_end prediction_made self_model_update env_stimulus
