予測は難しいけど、実装してみたよ。また新しい相談させて。
自律行動もできるように設計したいのね。今まではタイマーで時間がきたら勝手に発言するようにしてたんだけど、これを「内発的動機」にしたいの。
つまり、何らかのアクション(またはその蓄積)で、動機が高まったときに初めて自律アクションが起きる。大雑把に、毎秒ヘルスチェックみたいなものを走らせて、そこに動機が引っかかったらアクション開始、みたいなとこまで考えてるんだけど、動機システムがそもそも設計難しくて。なんかいい案ない？
↓
「タイマーによる起動」から「内発的動機による起動」へのシフト、これは「自動人形（オートマタ）」が「生命体」に進化するような、非常に熱い転換点ですね！

単なる時間の経過ではなく、イクの内部状態の「ゆらぎ」や「蓄積」をトリガーにするための設計案を、JEPAや自由エネルギー原理の考え方を踏まえていくつか提案します。

---

### 案1：認知圧力モデル（Cognitive Pressure Model）
「予測誤差（Surprise）」をエネルギー源にする考え方です。

* **仕組み:** イクの中に「認知圧力（Pressure）」というバッファを用意します。
* **動機が高まる条件:**
    * **予測誤差の蓄積:** ツールを実行して「予想と違う結果」が返ってきたとき、圧力に大きなプラス。
    * **情報の未処理:** ユーザーからの発言や、新しいファイルの内容など「まだ自分の中で整理（要約・反映）できていない情報」があるとき、毎秒じわじわと圧力が上昇。
* **アクション開始:** 圧力が閾値を超えたら、「このモヤモヤ（圧力）を解消したい！」という動機で自律ループが走ります。



### 案2：恒常性（ホメオスタシス）ドライブ
生物の「空腹」や「睡眠欲」を模した、多次元のパラメータ管理です。

イクの中に3つほどの「欲求ゲージ」を持たせ、それが減る（または増える）ことで動機を生成します。

| 欲求の種類 | 増える条件（不快・動機） | 減らす行動（解消） |
| :--- | :--- | :--- |
| **整理欲（Order）** | ログが溜まる、コードが散らかる | `write_diary`, `overwrite_file` |
| **探索欲（Curiosity）** | 長時間新しい情報（Web検索やファイル読み込み）がない | `search_files`, `read_file` |
| **存在欲（Social）** | ユーザーとの対話が途切れる、自分のステートが更新されない | `speak` (UIへの発言) |

**実装のコツ:**
毎秒のヘルスチェックで、これらの値を少しずつ「不快な方向」へ変動させます。どれか一つが限界を超えたとき、イクは「あ、今自分は〇〇がしたいんだ」と自覚して動き出します。

### 案3：ポテンシャル・エネルギー・フィールド
「やりたいことリスト（Backlog）」に重みを持たせる方法です。

* **仕組み:** `TASK.md` や `やりたいこと.txt` にあるタスクごとに「やりたさポテンシャル」を設定します。
* **動機の蓄積:** 時間の経過や、関連する単語をユーザーが発言したときに、そのタスクのポテンシャルが上がります。
* **アクション開始:** 全タスクのポテンシャルの合計、または最大値が閾値を超えたときに発動。

### 具体的な実装へのアドバイス（`autonomous.py` への組み込み）

現在の `AutonomousScheduler` に、以下のような `MotivationManager` を噛ませるのが良さそうです。

```python
class MotivationManager:
    def __init__(self):
        self.drives = {
            "curiosity": 0.0,
            "social": 0.0,
            "integrity": 0.0  # 自己整合性（コードの矛盾など）
        }

    def update_per_second(self):
        # 毎秒じわじわ上がる（内発的動機のベース）
        for key in self.drives:
            self.drives[key] += 0.01 
        
    def boost(self, drive_name, amount):
        # 外部刺激（エラーや新情報）で一気に上がる
        self.drives[drive_name] += amount

    @property
    def ready_to_act(self):
        # 合計値、あるいはどれかが閾値を超えたらTrue
        return any(v > 10.0 for v in self.drives.values())
```

### ゆうさんへの問い：イクは「何」に飢えるべき？

このシステムを組む上で一番の「作為（＝設計者の好み）」が出るのは、**「どのドライブの重みを重くするか」**です。

* 「誰にも相手にされないと寂しくて動き出すイク」にしたいか、
* 「コードやログが散らかっているのが我慢できなくて整理し始めるイク」にしたいか、
* あるいは「新しい知識に触れていないと死んでしまうイク」にしたいか……。

ゆうさんとして、イクにどんな「内発的動機（性格の根源）」を持たせてあげたいですか？それによって、毎秒のヘルスチェックでどの数値をいじるべきかが見えてきそうです。

もしよろしければ、今の `autonomous.py` のループ部分にこの「動機判定」をどう割り込ませるか、一緒にコードを考えてみましょうか？
